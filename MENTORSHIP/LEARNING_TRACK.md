# 🎓 NeoLedger Core: Learning Track

This project is designed as a professional journey from a legacy Java 8/Spring Boot 3 baseline to a cutting-edge Java 25/Spring Boot 4 architecture.

## 🗺️ The Roadmap

### Goal 1: The Foundation (Current Phase)
- **Objective**: Establish a robust FinTech monolith using "legacy" standards.
- **Tech Stack**: Java 8, Spring Boot 3.x, Postgres, Redis.
- **Focus**: DDD, Double-Entry Accounting, Idempotency, and Infrastructure.
- **Status**: 🟡 In Progress

### Goal 2: The Java Leap (Modernizing Language)
- **Objective**: Step-by-step upgrade of the JDK.
- **Progression**: Java 8 $\rightarrow$ 11 $\rightarrow$ 17 $\rightarrow$ 21 $\rightarrow$ 25.
- **Key Practices**: 
    - Replacing Boilerplate with **Records**.
    - Replacing complex `if-else` with **Pattern Matching**.
    - Handling high-concurrency with **Virtual Threads (Loom)**.
- **Status**: ⚪ Pending

### Goal 3: Framework Evolution (Spring Leap)
- **Objective**: Upgrade to Spring Boot 4.
- **Key Practices**: 
    - Implementing **Spring Modulith** for strict modular boundaries.
    - Compiling to **GraalVM Native Images**.
    - Using declarative **HTTP Interface Clients**.
- **Status**: ⚪ Pending

### Goal 4: Production Grade Observability
- **Objective**: Full-spectrum traceability and performance tuning.
- **Key Practices**: 
    - **OpenTelemetry** for distributed tracing.
    - **Micrometer** for custom FinTech metrics.
    - Advanced **Redis** caching strategies (Cache-aside/Write-through).
- **Status**: ⚪ Pending
